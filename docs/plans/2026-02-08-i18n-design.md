# RoboDesk i18n: Multi-Language Support

**Date:** 2026-02-08
**Status:** Approved

## Languages
- German (de) — default, current language
- English (en)
- Danish (da)
- French (fr)

## Architecture

### Translation object
Flat `translations` object at top of `robodesk.jsx`, keyed by locale code. ~90 keys with dot-notation namespacing (e.g. `nav.dashboard`, `form.save`).

### `t(key, params)` function
Simple lookup + interpolation. Placeholders use `{name}` syntax. Falls back to `de` if key missing in current locale.

### Storage
New key `robodesk-lang` in `window.storage`. Defaults to `de`.

### Language switcher
Dropdown in header next to theme toggle. Shows locale codes: DE, EN, DA, FR.

### Date formatting
Locale map: `de→de-DE`, `en→en-GB`, `da→da-DK`, `fr→fr-FR`. Used in `formatDate()` and `toLocaleDateString()` calls.

### What stays untranslated
- Relationship types (Professional, Personal, etc.) — user data
- Tag names — user data
- Contact field values — user data

## Translation Keys (~90)

| Prefix | Area | Keys |
|---|---|---|
| `nav.*` | Navigation tabs | 2 |
| `header.*` | Header bar, data menu | 10 |
| `toolbar.*` | Search, filters, sort | 10 |
| `bulk.*` | Bulk mode toolbar | 8 |
| `dash.*` | Dashboard sections | 13 |
| `detail.*` | Contact detail view | 15 |
| `form.*` | Contact form | 22 |
| `time.*` | Relative time display | 3 |
| `type.*` | Interaction types | 6 |
| `strength.*` | Relationship strength labels | 5 |
| `nudge.*` | Smart nudge messages | 6 |

## Implementation
1. Add `LANG_KEY` constant and `lang` state
2. Create German translations object (extract all strings)
3. Translate to EN, DA, FR via DeepL
4. Add `t()` function
5. Add language switcher to header
6. Replace all hardcoded strings with `t()` calls
7. Make `formatDate` locale-aware
